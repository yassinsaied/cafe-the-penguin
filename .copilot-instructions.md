# Copilot Instructions for Café The Penguin

This file configures how GitHub Copilot should assist with this Next.js project using integrated agent skills.

## Available Skills

### 1. Deploy to Vercel (`deploy-to-vercel`)
**When to use:** User asks to deploy, push live, create preview, or check deployment status.

**Key rules:**
- Always gather project state first: check git remote, `.vercel/` linking, CLI auth, teams
- Default to preview deployments unless user explicitly asks for production
- Ask for team selection if multiple teams exist
- Use `--scope <team-slug>` for team deployments
- Goal: get user into git-push auto-deploy setup

**Skill file:** `.agents/skills/deploy-to-vercel/SKILL.md`

---

### 2. React Best Practices (`vercel-react-best-practices`)
**When to use:** Writing/reviewing React components, Next.js pages, optimizing performance, data fetching.

**Key priorities (in order):**
1. **CRITICAL:** Eliminating Waterfalls (`async-*`)
   - Check cheap sync conditions before awaiting
   - Move await into branches where used
   - Use Promise.all() for independent operations
   - Use Suspense for streaming

2. **CRITICAL:** Bundle Size (`bundle-*`)
   - Import directly, avoid barrel files
   - Use statically analyzable paths
   - Conditional imports for heavy dependencies
   - Dynamic imports for code-splitting

3. **HIGH:** Server-Side Performance (`server-*`)
4. **MEDIUM-HIGH:** Client-Side Data Fetching (`client-*`)
5. **MEDIUM:** Re-render Optimization & Rendering (`rendering-*`)
6. **LOW-MEDIUM:** JavaScript Performance (`js-*`)
7. **LOW:** Advanced Patterns (`advanced-*`)

**Skill file:** `.agents/skills/vercel-react-best-practices/SKILL.md`

---

### 3. Composition Patterns (`vercel-composition-patterns`)
**When to use:** Building reusable components, handling boolean props proliferation, component libraries, React 19 APIs.

**Topics covered:**
- Avoid boolean prop explosion
- Compound components pattern
- Render props vs children
- Explicit variants over conditional logic
- Context for state management
- React 19 breaking changes (no ForwardRef)

**Skill file:** `.agents/skills/vercel-composition-patterns/SKILL.md`

---

### 4. React Native Skills (`vercel-react-native-skills`)
**When to use:** Building React Native/Expo components, mobile performance, animations, native modules.

**Note:** Currently not applicable (Next.js web project), but available if project expands to mobile.

---

### 5. View Transitions (`vercel-react-view-transitions`)
**When to use:** Adding page transitions, animating route changes, shared element animations, smooth UI updates.

**Use cases:**
- Smooth navigation between pages
- Enter/exit animations for components
- Directional (forward/back) animations

---

### 6. Web Design Guidelines (`web-design-guidelines`)
**When to use:** Reviewing UI, checking accessibility, auditing design against best practices.

## Usage Rules for Copilot

1. **Always check the relevant skill file** before responding to architecture/performance questions
2. **Reference specific rules** by their prefix (e.g., `async-parallel`, `bundle-barrel-imports`)
3. **Prioritize by impact:** CRITICAL > HIGH > MEDIUM-HIGH > MEDIUM > LOW
4. **For deployment:** Run the state-check script first, then propose the appropriate method
5. **For React code:** Apply best practices by category, starting with waterfalls and bundle size
6. **Always verify build** (`npm run build`) before pushing to ensure no regressions

## Quick Commands

```bash
# Check build locally
npm run build

# Push changes (auto-triggers Vercel deploy)
git push origin main

# Deploy to Vercel (if not git-linked)
vercel deploy --prod

# Check deployment status
vercel inspect <url>
```
